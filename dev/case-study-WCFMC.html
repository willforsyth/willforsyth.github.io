WhoCanFixMyCar | Our assumptions are as follows:

  • 1) User needs work done and has found us.
  • 2) User finds the garage profile.
  • 3) User doesn't want to, or doesn't know they can post a job but do like the look of the garage. There are no contact details available - so they exit, google the garage details and book direct.”

Ways to tackle the issue

  • Google analytic monitoring and study.
  • Run some online usability tests and give them a basic task to contact / organise an appointment with the garage.
  • Add in a form to ask the user if they have found what they want / what they are looking for on this page.
  • Carry out some interviews with customers to find out some key information.
    • How do they normally book an appointment with a garage
    • Do they know what they want when they contact a garage?
    • Do they know what the issue is with their car when they contact a garage. IE is it a unknow issue or do they need a service or MOT. If it’s a unknow issue would they want to diagnose the issue to get a rough idea of cost or would they be happy to book a time for a diagnosis?
    • Do they feel more comfortable speaking to a mechanic or the garage to discuss the issue?

From this we could get a better understanding of the reason for people visiting this page and how we can improve the whole garage experience. If they are visiting this page so they can find out more about the garage get some reviews and then find contact details to talk to someone how can we give them a better experience.

Without extra research we will base the rest of the process on the assumption off:

“The user would like to book an appointment with the garage."

User goals

  • As a driver I want to book my car in for an MOT / Service.
  • As a driver I have an unknown issue with my car I want to get it fixed as conveniently as possible.
  • As a driver I want to book an appointment for a garage near my home.
  • As a driver I want to find the details of a garage.

Sketches

I started of sketching out some user flows and rough ideas. Trying to work out any potential orders or issues the user will have.

Interaction flow

Once I had a solid idea I started to plan out in more detail how the user will input data and complete the booking.

"I tried to stick to the core user journeys, time permitting I would investigate these further."

Wireframes

This shows the user flows in more detail and how the interactions would work. Ideally I would go into more detail with the labeling and documentation. The click through prototype only takes in one flow ideally I would expand this to make a full prototype to test.

    Key features of new structure
  • Tap bar to access elements of form.
  • Estimate cost for slected services.
"If the garage has no way to show available times then the user could select any time and the garage would confirm the appointment."

Design

I then added some design styles to the wireframes and started to build up the steps in a little more detail. I have changed some of the colours slightly to modernise the colour pallette. Providing a gif helps to demonstrate how the interactions could work.

    Key features of new design
  • Refresh of colour and styles used.
  • Introduction of illustrations.
“If you were looking for a brand refresh in the future you may want to have bespoke illustrations created depending on the direction of new brand styles.”

This demonstrates the design process I like to follow. Ideally we would spend a lot more time gathering research, then moving on to testing the proposed ideas with users as often and quickly as possible.
