UX Process

The process starts in one of two places but follows the same structure. If you have a product already then testing and analysis can begin on a live product or if you are in the development phase defining a prototype and and testing that will help reduce future issues. There are several ways we can do this:

USABILITY TESTING

Live usability testing with real users. Find their goals, behaviours, pain points and mental models.

CONTEXTUAL INQUIRY

Interviews and testing in the users environments.

EYETRACKING & SCREEN TRACKING

Recording users activities on your site and create reports and gain vital insights

ANSWER QUESTIONS

Using software to ask users questions using surveys and on page questionnaires.

Competative research

Detailed research onto competitors and indirect competitors, how they have tackled similar problems, gained revenue streams and how they have gained competative advantages.

DATA ANALYSIS

Using UX software and in-depth analytics research I can gather data on where improvements need to be made.

Using the research I can find triggers, barriers and pain points, entry points and actions such as conversion access and exit. Find your users goals and motivations. To do this the best solution I have found is to involve the full team and split out the research into several sections depending on the product. Outlining all pain points, triggers and making sure everyone is on the same page as to what the problems are.

I can also use Data Analysis at this stage to further access the possible issues. Using UX software and in-depth analytics research I can gather data on where improvements need to be made.

Analysis

After we have completed the research phase its time to analyse the data to gather insights find patterns and discover solutions.

Affinity Diagramming & Card sorting

Write all observations down on post-its and group into actions, triggers, pain points and barriers. Identify patterns.

User Journey Maps

Map out the user journey experienced in the usability test.

Prioritising Matrix

Prioritise the improvements over a time period checking against technical feasibility, design feasibility, and impact on user.

Personas

Set the context, define the users goals, behaviours and look into how detail about who the user is and how they think.

Design

Once I have a firm set of problems to solve the next stage is planning out user flows wireframing solutions and reworking information architecture. Once we have the solutions solved ( in our opinion ) its time to create a prototype and re test the solution with users to ensure our opinion is gratified and correct.

Information architecture

Extensive information architecture planning and scoping of the site or app structure

Story Boarding

Story board the proposed journey for the user. If we can, test this on users at this stage.

Interaction flows

Detailing every user and interaction flow, and creating test prototypes where applicable. Outlining in diagrams or creating small prototypes

Wireframes

Wirefame as much of the site/app as we need to create a prototype that will demostrate the user journey of as many processes as possible. Defining where content will live etc.

Prototype

Development of a prototype to test on users, either in marvel or wuth code.

User testing

This is where we ideally need to test on the users again. Completeing previous steps as required and iterating.
