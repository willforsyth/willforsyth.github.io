Research

The process starts in one of two places but follows the same structure. If you have a product already then testing and analysis can begin on a live product or if you are in the development phase defining a prototype and testing that is essential. There are several ways to do this:

Usability testing

Live usability testing with real users. Find their goals, behaviours, pain points and mental models.

Contextual enquiry

Interviews and testing in the users environments.

Eyetracking & Screen Tracking

Recording users activities on your site and create reports and gain vital insights

Answer Questions

Using software to ask users questions using surveys and on page questionnaires.

Competitive research

Detailed research into competitors and indirect competitors, how they have tackled similar problems, gained revenue streams and how they have gained competitive advantages.

Data Aanalysis

Using UX software and in-depth analytics research I can gather data on where improvements need to be made.

This research phase may be repeat multiple times throughout the process.

Analysis

Using the research I can find triggers, barriers and pain points, entry points and actions such as conversion access and exit. Find your users goals and motivations. To do this the best solution I have found is to involve the full team and split out the research into several sections depending on the product. Outlining all pain points, triggers and making sure everyone is on the same page as to what the problems are.

Affinity Diagramming & Card Sorting

Write all observations down on post-its and group into actions, triggers, pain points and barriers. Identify patterns.

User Journey Maps

Map out the user journey experienced in the usability test.

Prioritising Matrix

Prioritise the improvements over a time period checking against technical feasibility, design feasibility, and impact on user.

Personas

Set the context, define the users goals, behaviours and look into how detail about who the user is and how they think.

After the analysis we should know more about what we need to create to allow the user to acheive their goals on the product. Next is planning these components and journeys out.

Design

Once I have a firm set of problems to solve the next stage is planning out user flows, stroryboarding, wireframing solutions and reworking/creating information architecture. Once we have the solutions solved ( in our opinion ) it's time to create a prototype and re-test the solution with users to ensure our opinion is gratified and correct.

Information architecture

Extensive information architecture planning and scoping of the site or app structure

Story Boarding

Story board the proposed journey for the user. If we can, test this on users at this stage.

Interaction flows

Detailing every user and interaction flow, and creating test prototypes where applicable. Outlining in diagrams or creating small prototypes

Wireframes

Wireframe as much of the site/app as we need to create a prototype that will demonstrate the user journey of as many processes as possible. Defining where content will live etc.

Prototype

Development of a prototype to test on users, either in marvel or with code.

User testing

This is where we ideally need to test on the users again. Completing previous steps as required and iterating.

UI Design

After we have a full scoped out product with all the user needs addressed and signed of the last stage is adding the branding and styles to the layouts. I use Sketch and other programmes such as principle of the framer.js library bring the designs to life if reuqire to show state changes etc.
