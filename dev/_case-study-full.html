Money Guru | Case study

Money guru is a comparison site that went from 0 - 1 million users in its first 6months and became the fasted growing comparison website in the UK in 2017. This is how we applied a user centric approach to research, design and iterateing the product ensuring we continually monitor and improve the experience.

Thankfully the talented team I work with created the brand, the mood and tone the copy, marketing campaigns and advertising to support the product. All we had to do was create the product ;-).

We started working to get a MVP produced as early as possible.

0.1 | UX Research

The research phase of a new product is always the most difficult, it takes more time to understand the user and is difficult to get usability testing completed on a product. So we decided early to get a MVP out as quickly as possible and then develop the product based on concrete research.

What we used

  • Competitor research confused.com, gocompare.com etc
  • Analytics research they had an old site that we were going to use some data from and the audience would be similar so we could gather some data to analyse.
  • Surveys and questinnaires we added hotjar to the site to gather as much data we could on the users behaviours.
  • User stories we conducted over 100 user stories to delve into what the users needs were.
  • User personas we then selected some key user personas to study more and find out what issues and goals they have.

0.2 | UX analysis

We had started to gather some useful research, so the next stage was to analyse the data and start to think off solutions for the users we have defined.

What we used

  • User story maps helped us to view the whole project and detail how each persona would acheive their goals.
  • Card sorting helped us to prioritise these features.
  • Affinity diagrams ...

0.3 | UX design

When we had prioritised the features it was time to revist the personas and work out user flows for each persona I looked at the site map and information architecture, planning out all aspects of the layout including interactions and user flows. Each wireframe is a detailed lists of all aspects of how the user interacts and users the site. Including users flows and different states.

What we used

  • User flows helped us detail each interaction for each personas journey to its completion and discover how they acheived their goals.
  • Interaction design helped us detail each interaction for each personas journey to its completion and discover how they acheived their goals.
  • Wireframes helped us detail each interaction for each personas journey to its completion and discover how they acheived their goals.

0.4 | Prototype

Once I had a solid structure I created a prototype which we could then test.

What we used

  • Sketch helped us detail each interaction for each personas journey to its completion and discover how they acheived their goals.
  • Marvel app helped us detail each interaction for each personas journey to its completion and discover how they acheived their goals.

0.5 | UI design

Using the style guide and branding guidelines high fidelity designs were created.

User testing

User testing was always going to be a large part of this project and in my opinion the most important part. It is esential to make sure the assumptions made throughout the project are correct. So a extensive user testing session was completed to carry brand feedback and to user test the inital layouts.

What we found out

  • Smart search the smart search was not offering enough benefits and was hard to understand.
  • Navigation the navigation need some tweaking to offer direct access from all over the site.

Our involvement in phase 1 was over.

The site now went into to development to get a MVP up as early as possible. We needed to carry out some more user testing and we had parked a lot of key features.

We started to look forward to phase 2. In the background the marketing and socail media team were working hard.
